User Account Block

This dialog box may be used to create the QPL code to create one user account. This code will be inserted into your program at the current cursor position.

User name Enter the name a user will use when logging into the project web site. The user name must only contain letters and numbers, and must be between five and 50 characters long. User names are not case-sensitive. If left blank, the user name "USER#" will be automatically created when the job.sql script is run to create the web site.
Password Enter the password a user will use when logging into the project web site. The password must only contain letters and numbers, and must be between five and 50 characters long. Passwords are not case-sensitive. If left blank, a random password number will be automatically created when the job.sql script is run to create the web site.
Group The Group field is used to assign a Normal user to a group. This allows each Normal user in that group to access the same questionnaires. If left blank, the user is not assigned to any group. The Group field is not used with Manager or Administrator accounts.
Name Enter the full name of the account holder.
Email Enter the e-mail address of the account holder (Ex. myname@myhost.com).
Expire Enter the date that a Normal account should expire. The account will expire at midnight on this day. This field does not affect Manager and Administrator accounts. These accounts never expire.
Phone Enter the phone number of the account holder.
Comments Enter any other comments you wish to make about this account holder. These comments will appear on the user account page when an Administrator accesses the web site.
Status Select the type of account:

Closed - Account is created but may not be used.

Normal - Access the questionnaire.

Manager - Access the administrative pages to view project statistics and reports.

Administrator - Has Manager rights plus access to update user accounts and content analysis pages.
