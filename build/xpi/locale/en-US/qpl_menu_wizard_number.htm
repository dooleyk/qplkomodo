Number Question

This dialog box may be used to create the QPL code for one number question. This code will be inserted into your program at the current cursor position.

Name The name of a question must start with a letter, contain only letters and numbers, and be from one to sixteen characters long.

If a name is not specified, then a name will be created automatically. (Name will be Q# if visible, U# if visible but not numbered, or H#if not visible.)
Stat label If a label is entered here, then this will be used as a variable label for this question when the SAS or SPSS program is generated instead of the question text.
Align text Define how the question text should be aligned.

Left - Left justify question text
Center - Center question text
Right - Right justify question text
Inherit - Inherit setting from the previous question
Align question Define how the question should be aligned vertically when used inside a COLUMNS layout. This setting has no effect on questions in normal or MATRIX layout.

Top - Top-align question
Middle - Center-align question
Bottom - Bottom-align question
Baseline - The baseline is the "imaginary line" which most letters "sit" on, in a line of text. The baseline value sets the row so that all the table data share the same baseline. Often this has the same effect as the bottom value. However, if the fonts are in different sizes, baseline looks better.
Inherit - Inherit setting from the previous question
Bold text Set question text to be displayed using a bold font.

On - Use bold font.
Off - Do not use bold font
Inherit - Inherit setting from the previous question
Bold text Set question text to be displayed using a bold font.

On - Use bold font.
Off - Do not use bold font
Inherit - Inherit setting from the previous question
Width Set the width of the question text area as the number of pixels. Leave field blank to inherit this setting from the previous question. Enter "D" to reset the width to the current system default. This value will vary according to whether the question is in a normal, matrix, or columns layout.
Question Enter the text of your question here. Each line must be less than 500 characters long. You do not need to be careful where you break your lines; the text will be re-wrapped when it is displayed in the web questionnaire form. You can create a paragraph break by entering a blank line. (See the "Text Line Formatting" section below, and the QPL Help page, "QPL Language Syntax," for additional ways to format your question.)
Answer option Use this option to specify the maximum number of characters that may be typed by the respondent into the number field on the web questionnaire form. For example, if you want to limit the respondent's answer to ten digits, you should enter "10" here.

You may instead use this option to copy your field size or format setting (see below) from another number question that you have already written in your questionnaire program. To do this, enter the name of the other number question here. (If you enter the name of another question here, you should not enter a format or default answer below.)
Next Entering the name of a question here will insert a hypertext link to that question at the bottom of this question. The respondent may click on this link to jump to the question, whether or not it happens to be on the same page. No link is created if this left blank.
Display (Define Target) Enter a DISPLAY name for this question. You can trigger the display of this question by linking it to an answer in a MULT or CHECK question by entering the DISPLAY command with this name after the answer text in that question. This question will be hidden until the respondent selects that answer on that question.
Required Check this option to make the answer to this question required. The input field will be displayed on the form with a light pink background if the respondent has not entered an answer. Once the respondent has entered an answer and moved to another question, the light pink background will change to the normal white background.
Format The format option lets you put a label on the left or right side, or both sides, of the input field on the web questionnaire form and specify the number of decimal places. Enter "#" characters to indicate the number of digits that may be entered and any other characters will be your label. (Ex. "$##.##", will create a four-digit input field with two decimal places and with the "$" displayed on the left-side of the input field.) If you use a format to set your field size, you should not enter a number at the Answer option above to set the field size.

You may instead use the format to just enter the number of decimal places. By default, if no format is used, the number of decimal places is set to zero for integers. The total field size set at the Answer option must be greater than the number of decimal places you set at the Format option.
Default Normally, the number input field defaults to a blank answer. You may enter a different default answer here. All respondents will see the same default answer. (See the QPL Help page, "Preloading User Accounts and Data," if you want each respondent to see a personalized default answer.)
Rank group A RANK group may be used to turn on automatic ranking rules for two or more NUMBER questions. This is useful when you want a respondent to put a list into a rank order. Here, the question text for a NUMBER question is the item to be ranked, and the value entered is the order number (which may be 1 through the to the total number items (i.e., questions) in your list). The automatic rules will warn the respondent if they use a number out of this range, or if the use the same number more than once. Blank responses are not considered to be errors. These error messages are updated when the respondent moves the cursor out of a NUMBER question input field. Enter the name of the group using the RANK command inside each question that you wish to include within the ranking group. (You may instead use the RANK command with a name before the first question in your series and then a RANK = OFF command after the last question to mark a block questions in your ranking group.) All the NUMBER questions in a RANK group must be on the same page of the questionnaire. You may also display an general error message to the respondent by putting the name of the RANK group in the text of question, delimited by asterisk characters.
Sum group A SUM group may be used to give the respondent a running tally for two or more NUMBER questions. This is useful, for example, when you want the respondent's answers to total to a certain value, such as 100%. The total information is automatically updated when the respondent moves the cursor of a NUMBER question input field. Enter the name of the group using the SUM command inside each question that you wish to include within the tally group. (You may instead use the SUM command with a name before the first question in your series and then a SUM = OFF command after the last question to mark a block questions in your tally group.) All the NUMBER questions in a SUM group must be on the same page of the questionnaire. Display the total value to the respondent by putting the name of the SUM group in the text of question, delimited by asterisk characters.
